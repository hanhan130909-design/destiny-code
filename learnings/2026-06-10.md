### 3.3 学到了什么

#### 工程经验

1. **「一鸡多吃」内容策略**
   - 1 篇博客 → SEO 文章 + FB 帖 + IG 帖 + Newsletter + RSS + Pinterest Pin
   - 一次创作，6 个渠道分发，ROI 最大化

2. **API 接入三原则**
   - 优先用免费 API 自动化
   - 付费 API 找免费替代（如 Buffer 替代 Pinterest API）
   - Token 安全第一：永久 token 必须加密存储

3. **部署即监控**
   - Vercel 项目可能因长期不活跃被清理
   - 必须设置健康检查 + 告警（UptimeRobot 或 cron health check）

#### 战略认知

4. **赛道认知刷新：BaZi AI 不是蓝海**
   - 6 个月前只有 3-4 个竞品，现在至少 10+
   - 但所有新竞品都还没开始变现 → 这是我们的窗口

5. **AI 解读不是加分项，是入场券**
   - 2026 年的用户预期：输入生日 → AI 立即给出解读
   - 等待人工撰写报告的模式已过时

6. **先发定价优势**
   - 新竞品全部免费，用户对「免费」已脱敏
   - $29 定价 + 专业品牌 = 可感知的价值信号

   - 竞品普遍只做搜索/SEO 单一渠道
