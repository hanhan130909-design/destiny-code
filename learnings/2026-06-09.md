## 一、社交媒体自动化实战经验（今日产出）

### 1.1 Facebook Graph API 接入

**踩坑记录**：
- `pages_manage_posts` 权限需在 Graph API Explorer 中勾选，不在默认列表里
- 短期 token → 长期 token → Page token 三步兑换
- App Review → Permissions → 需先申请权限才能在 Explorer 中看到
- 直接 `/{page_id}?fields=access_token` 可绕过 OAuth 流程拿 page token
- Page token 永不过期（除非用户改密码或移除授权）

**自动化方案**：
- 脚本从 29 篇博客中随机抽取未发布过的文章
- 截取 snippet + 链接作为帖子内容
- 每天 10:00 自动发，log 记录避免重复

### 1.2 Instagram Graph API 接入

**踩坑记录**：
- IG 账号必须是 Business/Creator 类型
- 需在 FB Page 设置中关联 IG 账号
- 发帖流程：创建 media container → 等待 FINISHED → publish
- 图片必须是公网可访问 URL（不能直接上传文件）
- FAL AI 图片生成失败时用 Python Pillow 生成 OG 图兜底

**自动化方案**：
- 复用 FB 的 page token
- 每天 14:00 发，与 FB 错开时段
- 使用同一张 OG 图（后续可做多图轮换）

### 1.3 Pinterest API（失败经验）

**结论**：Pinterest API v5 需要 Standard 付费计划
- Trial 版只有 `pins:read` 等读权限，无 `pins:write`
- 错误码 3："consumer type is not supported"
- 免费方案：Buffer → Pinterest 直连（绕过 API）

### 1.4 Hashnode API（失败经验）

**结论**：2026年5月起 Hashnode GraphQL API 转为付费
- 免费 API 已退役，读写均需 Pro 计划
- 替代方案：RSS Feed 一键导入

## 三、本周学到的增长原则

1. **「一鸡多吃」**：一篇博客 → SEO 文章 + FB 帖 + IG 帖 + Newsletter + RSS 分发
2. **API First**：优先用 API 自动化，手动操作的渠道优先级降低
3. **付费 API 即时放弃**：不要为 API 付费，找免费替代（如 Buffer 替代直连 API）
4. **反向链接是核心**：Dev.to + RSS + 社交帖子中的链接 → 提升域名权重
5. **Token 安全**：Graph API token 永久有效，需安全存储；Token 泄漏 = 账号接管
